# Pastry 产品说明

本文档记录 Pastry 当前代码已经实现的产品行为，用于产品理解、验收和后续迭代对齐。不包含尚未落地的路线图。

## 产品定位

Pastry 是一个 macOS 26+ 剪贴板历史管理器。它常驻菜单栏，在后台监听系统剪贴板变化，将文本、链接、图片、文件、RTF、HTML 等内容保存为可搜索、可筛选、可收藏的历史卡片，并通过全屏半透明 overlay 面板快速找回和粘贴。

核心体验是：

1. 用户在任意应用中复制内容。
2. Pastry 识别内容格式、来源应用和时间，并保存到本地历史。
3. 用户通过全局快捷键或菜单栏打开面板。
4. 用户搜索、筛选、选择卡片，并粘贴、复制、拖拽、预览、打开或分享。

## 入口与主流程

### 菜单栏

Pastry 以 LSUIElement 菜单栏应用运行，不占用 Dock。菜单栏图标提供两个入口：

- 左键：打开或切换剪贴板面板。
- 右键菜单：打开剪贴板、查看统计和存储占用、清空历史、打开关于窗口、打开设置、检查更新、退出应用。

菜单栏「清空历史」会删除非收藏记录并保留收藏记录。如果清空后历史为空，会同步清空系统剪贴板。

### 全局快捷键

默认快捷键是 `Command + Shift + V`。用户可以在设置中重新录制快捷键，修改后会立即重新注册全局热键。

快捷键触发后，Pastry 显示一个不激活前台应用的全屏透明 overlay。面板使用底部托盘布局，宽屏下横向展示卡片，窄屏下纵向展示卡片。

### 粘贴流程

点击已选中的单张卡片会触发粘贴。键盘确认或快捷粘贴会先将对应内容写回系统剪贴板，再模拟 `Command + V` 粘贴到原目标应用。

多选粘贴时，会将选中的内容合并为一个多行文本写回剪贴板。文件和图片会尽量以文件 URL 形式参与拖拽或写入。

## 支持的剪贴板内容

Pastry 当前支持以下内容类型：

| 类型 | 采集与展示 |
|---|---|
| 文本 | 保存纯文本内容，卡片显示最多 7 行预览，数据库保留完整内容。 |
| 链接 | http/https 链接会被标记为 Link；多行链接会显示为多链接列表。 |
| RTF | 保存纯文本内容，同时保留原始 RTF 数据用于写回和预览。 |
| HTML | 保存纯文本内容，同时保留原始 HTML 数据；会尝试解析 HTML 中的有序图文段。 |
| 图片 | 保存图片缩略图和原图映射，卡片显示图片预览；可带截图文字注释。 |
| 文件 | 保存本地文件路径，支持单文件和多文件列表，卡片异步加载图标或图片缩略图。 |
| Handoff | 识别 Universal Clipboard 来源，并在卡片底部标记为来自其他设备。 |

读取优先级按更具体的类型优先：文件 URL、网页 URL、图片、HTML、RTF、文本。对于微信/QQ 的自定义富文本，Pastry 会尝试从 `TencentAttributeStringType` 中提取文本内容。

## 面板 UI

面板由半透明托盘和固定尺寸卡片组成。顶部工具栏包含搜索、筛选、All/Favorites 标签和设置入口。

### 卡片结构

每张卡片包含：

- 顶部类型栏：显示格式图标和类型文字，背景使用来源应用提取出的主题色。
- 应用图标：显示来源应用图标；Handoff 内容显示设备联动图标。
- 内容区：按类型渲染文本、链接、图片、文件、HTML 图文或多链接列表。
- 底部信息：显示相对时间、来源应用或 Handoff 标记；收藏项额外显示 pin 图标。

链接卡片在网络预览开启后会尝试加载标题、描述和缩略图。没有预览数据时，会用 URL host 和路径降级展示。

### 选择与键盘行为

面板支持：

- 单击选择卡片，再次点击已选卡片进行粘贴。
- `Command` 点击进行多选。
- `Shift` 点击或方向键扩展选择范围。
- 方向键移动当前选择。
- `Enter` 粘贴当前选择。
- `Command + 1...9` 快速粘贴前 9 张卡片。
- `Command + A` 全选当前可见卡片。
- `Delete` 删除选中卡片，并弹出确认。
- `Esc` 关闭面板。

按住 `Command` 时，前 9 张卡片右下角会显示数字角标。

### 搜索与筛选

搜索框支持对内容进行全文搜索。当前实现优先使用 SQLite FTS 搜索内容和链接标题，同时会将来源应用名称匹配的结果合并进来。

筛选支持：

- 来源应用筛选。
- Handoff 筛选。
- 类型筛选：Text、Image、File、RTF、HTML 等。
- 时间筛选：全部、今天、昨天、本周、上周、最近 30 天。
- All / Favorites 标签切换。

面板关闭或重新打开时会清空临时搜索和筛选状态。

## 卡片操作

右键卡片会打开系统风格菜单。根据内容类型和文件存在性，菜单项会自动启用或禁用：

- 收藏 / 取消收藏。
- 复制：单选时写回原内容；多选时合并为多行文本。
- 打开：链接用默认浏览器打开，文件用默认应用打开，多链接或多文件会逐个打开。
- 打开方式：对可打开内容展示系统可用应用，并支持手动选择其他应用。
- 在 Finder 中显示：仅文件类内容可用。
- 预览：使用 Quick Look 风格预览；文本、RTF、HTML、链接会生成临时预览内容。
- 分享：调用系统分享服务。
- 删除：删除历史记录，不主动清空系统剪贴板。

面板内批量删除会删除选中记录；如果删除后历史为空，会同步清空系统剪贴板。

## 拖拽

卡片支持拖拽到其他应用。

单张拖拽：

- 链接优先暴露 URL，同时注册纯文本。
- 图片优先使用原图文件路径。
- 文件使用文件 URL。
- 文本、RTF、HTML 使用文本内容。

多选拖拽：

- 使用单个拖拽项，避免系统为多个 dragging item 自动叠加数量角标。
- 文本载荷会按卡片顺序合并为多行。
- 全部为链接时，同时记录链接 URL 列表。
- 存在文件或图片时，同时记录存在于磁盘的文件 URL。

拖拽开始时面板会临时隐藏并允许鼠标事件穿透，避免 overlay 阻挡目标应用接收拖拽。

## 设置

设置窗口包含 General、Shortcut、Security 三个页签。

### General

- 语言：跟随系统、简体中文、英文。
- Launch at Login：通过 `SMAppService` 管理开机启动。
- 声音：控制复制和粘贴提示音。
- 历史保留：最大条数可选 100、500、1000、2000、5000；保留周期可选永不、7 天、30 天、90 天、1 年。
- 辅助功能授权状态：显示当前是否已获得 Accessibility 权限，并提供跳转系统设置入口。
- 关于：显示当前版本并打开 About 窗口。
- 清空全部记录：删除全部历史记录并清空系统剪贴板。

### Shortcut

用于录制全局面板快捷键。录制开始时会临时注销旧快捷键，保存或取消后重新注册。

### Security

- Accessibility 权限状态。
- 链接预览网络访问开关。
- 排除应用列表：被排除应用中的复制内容不会写入 Pastry 历史；设置页支持添加和移除应用。

## 隐私与安全

Pastry 的历史数据默认保存在本机应用数据目录。

当前安全策略包括：

- 数据库使用 SQLCipher 全库加密。
- 数据库密钥优先使用数据库旁的 `.key` 文件保存；旧版本 Keychain 密钥只作为迁移来源。
- 检测到旧明文数据库时，会尝试迁移为加密数据库。
- 识别 `org.nspasteboard.ConcealedType` 时跳过记录，用于避免保存密码等隐藏剪贴板内容。
- 支持按 bundle id 排除指定应用。
- 链接预览网络访问默认关闭，需要用户在 Security 中开启。
- 远程资源只允许 https，并阻止 localhost、`.local`、私有网段、链路本地地址和组播地址。
- 链接预览限制 HTML 响应大小为 2 MB，图片响应大小为 5 MB。

Accessibility 权限主要用于跨进程粘贴模拟和更准确的来源应用识别。没有权限时，部分粘贴或来源识别能力会降级。

## 数据与保留策略

剪贴板记录存储在 `clips.db` 中，核心字段包括内容、类型、来源应用、时间戳、收藏状态、展示计数、Handoff 标记、原始格式数据、链接标题和去重 key。

保存逻辑：

- 新内容写入数据库后会刷新面板列表、统计和可筛选应用列表。
- 相同 dedup key 的历史记录会被删除并以新时间戳重新插入，相当于置顶最新来源。
- 5 秒内连续出现的相同内容会被视为快速重复并跳过。
- 面板内存列表默认加载最近 500 条，数据库可按保留策略保存更多。
- 数据库触发器有 50,000 条的兜底上限，防止异常增长。
- 保留策略会跳过收藏记录，避免收藏内容被自动清理。

图片内容由 `ImageCacheManager` 管理缩略图和原图映射。文件类卡片只保存路径；如果原文件被删除，卡片会显示缺失状态。

## 更新与发布

正式构建的菜单栏包含「检查更新」入口。开发构建中该入口会被禁用，避免误触发正式更新流程。

发布流程由仓库脚本处理：

- `./deploy.sh`：构建、签名并启动开发版。
- `./release.sh [version]`：测试、release 构建、签名、打包 DMG 并烟测。
- `./release.sh [version] --publish`：额外推送 tag 并创建 GitHub Release。

## 验收场景

可以用以下场景验证当前产品行为：

1. 复制普通文本、链接、图片、文件和 HTML 富文本，确认面板生成对应卡片。
2. 用 `Command + Shift + V` 打开面板，确认 All/Favorites、搜索、筛选和设置入口可用。
3. 搜索内容和来源应用名，确认结果能命中。
4. 收藏卡片后执行菜单栏清空历史，确认收藏记录保留。
5. 对链接、文件、图片、文本分别右键，确认菜单项启用状态符合内容类型。
6. 多选几张卡片后复制、粘贴和拖拽，确认内容按多行合并或按文件 URL 传递。
7. 关闭链接预览网络访问时复制链接，确认不会主动抓取远程预览；开启后确认安全策略允许的 https 链接可加载预览。
8. 将某个应用加入排除列表后，在该应用中复制内容，确认不会新增历史。
9. 调整历史保留数量或时间后，确认非收藏旧记录会被清理，收藏记录仍保留。
10. 清空全部记录后，确认历史列表和系统剪贴板均被清空。
